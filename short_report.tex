\begin{abstract}
This report summarizes the current DSGD pipeline built on top of rules induced by FOIL and RIPPER. The earlier project baseline used an internal STATIC rule source built from automatically generated single literals and short conjunctions; the current pipeline instead induces readable rules first, freezes that ruleset, and only then learns evidential masses over the same fixed rules. Here, ``frozen ruleset'' means that rule induction is completed first and the later mass-learning stage does not change which rules exist. The present benchmark aggregates one fixed train/test split per dataset across five training seeds. That makes it stronger than a one-run snapshot, but it still should not be read as a split-robust generalization claim. Within that limited reading, learned evidential aggregation often improves over the raw rule aggregators built from the same frozen ruleset, while Random Forest remains the strongest overall external reference.
\end{abstract}

\section{Benchmark Setup}
The maintained pipeline has two stages. First, FOIL or RIPPER induces a readable candidate rule pool, and post-growth shaping filters it for quality, redundancy control, and class balance. Second, the final ruleset is frozen and passed to the evidential learner, which only adjusts rule masses; it does not invent new rules during mass training.

The runtime protocol is now explicit. Standard CLI runs use the user-provided seeds and test size, while the frozen paper protocol is an opt-in mode triggered by \texttt{--paper-mode}. That avoids silently overriding custom benchmark settings.

The report should also be explicit that these are project-specific FOIL/RIPPER pipelines rather than untouched off-the-shelf baselines. The present benchmark therefore compares aggregation methods on top of the repository's induced-and-shaped rule pools, not native ordered-rule prediction from an external FOIL or RIPPER package.

This separation matters for interpretation. FOIL and RIPPER determine which rules exist, while the DSGD layer determines how much evidential weight each fired rule receives when predictions are fused. The current raw baselines are defined on the same frozen ruleset: \texttt{first\_hit\_laplace} means ``use the first fired rule in the ordered list and read its Laplace-smoothed class estimate,'' while \texttt{weighted\_vote} aggregates all fired rules heuristically without learned evidential masses.

\section{Hard Cases}
To diagnose where learned aggregation may help, we defined a hard case as a held-out test sample that satisfies at least one conflict-oriented rule criterion inside the same benchmark split. All hard-case criteria are computed from fired rules of the frozen ruleset for the same dataset/inducer pair. A sample enters the hard-case subset if at least one of the following is true: multiple fired rules support different labels, the number of fired rules is unusually high, the weighted-vote margin is low, the true class has minority support against a competing class, or exactly one true-class rule is opposed by many wrong-class rules.

The hard-case tables are diagnostic and descriptive, not a standalone causal argument. In the current snapshot, some dataset/inducer pairs show sizable improvements over weighted vote on that subset, for example \texttt{adult/RIPPER} (Accuracy 0.8354 vs.\ 0.7882), \texttt{gas\_drift/RIPPER} (0.8746 vs.\ 0.6311), and \texttt{magic-gamma/FOIL} (0.8598 vs.\ 0.7688). Those results support only the narrower statement that learned masses can help on some conflict-heavy samples built from the same frozen ruleset.

The same analysis also shows limits. Some difficult pairs coincide with long or overlap-prone rule bases, and very small already-sharp rule bases leave little room for DSGD to help. This short report, however, does not isolate those factors well enough to make a stronger mechanistic claim about rule diversity or overlap.

\section{Illustrative Inspection Example}
The inspection view is best placed after the benchmark sections, because it is an explanatory aid rather than a separate source of evidence. In one income-style tabular sample, 11 rules fire at once and support conflicting labels. The inspection export shows the fired rules, their mass vectors, a merged condition summary, and the final fused mass vector.

That output is easier to read than a long unordered list of fired literals from the earlier internal STATIC pool. The claim should stay narrow: induced FOIL/RIPPER rules give the inspection tooling a cleaner rule-level object to display. The example supports only that readability-of-display point; it does not establish a broader comparative interpretability result, it does not prove why one aggregation strategy wins on average, and the merged combined condition is still an inspection-oriented summary rather than a logically exact replacement for the full fired ruleset.

\section{Conclusion}
The current supervisor-safe conclusion is limited and concrete: on the present fixed-split five-seed frozen-rule benchmark, learned evidential aggregation on RIPPER/FOIL-induced rules is often competitive with or better than raw rule aggregation on the same ruleset. RF remains the strongest overall reference. The right framing is therefore ``measured benchmark of a frozen-rule evidential pipeline'' rather than ``best overall classifier,'' ``proof of a general rule-diversity mechanism,'' or ``evidence of a general interpretability advantage.''
